## レポート作成時の注意

1. **必ず Phase 0 から書く**。「どの層を扱うか」の宣言なしに本論に入らない。
2. **未確定は未確定と書く**。憶測を確定であるかのように書くと、後段の意思決定が歪む。
3. **一貫性チェックは飛ばさない**。これが本スキルの最大の付加価値。
4. **テーブル / 図 / コードブロックを多用する**。長文より構造化された情報のほうが読まれる。
5. **「次のアクション」を必ず書く**。レポートは action item を生まなければ意味がない。
