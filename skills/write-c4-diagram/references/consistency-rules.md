# レベル間整合性ルール

複数レベルの図を作成した後に、このファイルの手順で整合性を機械的に検証する。

## 命名一致マトリクス

| 要素 | L1 | L2 | L3 | L4 |
|---|---|---|---|---|
| Person | `Person(id, "名前", "説明")` | 同一（名前・説明の完全一致） | — | — |
| System | `System(id, "名前", ...)` | `System_Boundary(id, "名前")` | — | — |
| External | `System_Ext(id, "名前", ...)` | 同一（名前・説明の完全一致） | 同一 | — |
| Container | — | `Container(id, "名前", ...)` | 外部参照: `Container(id, "名前", ...)` | — |
| ContainerDb | — | `ContainerDb(id, "名前", ...)` | 外部参照: `ContainerDb(id, "名前", ...)` | 外部参照 |
| Component | — | — | `Component(id, "名前", ...)` | 外部参照 |

## ズームイン検証

```
L1 の System()       → L2 の System_Boundary() に展開
L2 の Container()    → L3 の Container_Boundary() に展開
L3 の Component()    → L4 で詳細展開
```

## 検証チェックリスト（クロスレベルレビュー時に実施）

**L1 → L2 の整合性:**
- [ ] L2 の `System_Boundary` 名が L1 の `System` 名と完全一致しているか
- [ ] L2 の全 `Person` の名前・説明が L1 と完全一致しているか
- [ ] L2 の全 `System_Ext` の名前・説明が L1 と完全一致しているか

**L2 → L3 の整合性:**
- [ ] L2 の全 `Container` が L3 の外部参照として漏れなく出現しているか
- [ ] 展開対象のコンテナが `Container_Boundary` として（同じ名前で）定義されているか
- [ ] L3 で追加した外部参照 Container の名前が L2 と一致しているか

**L3 → L4 の整合性:**
- [ ] L3 の展開対象コンポーネントが L4 で正しく参照されているか
- [ ] L4 のクラス属性・メソッド名がソースコードと一致しているか

**関係線の一貫性:**
- [ ] L2 で A → B の関係があれば、L3 でも A の内部コンポーネント → B の関係が存在するか
- [ ] 関係の方向（呼び出し元 → 呼び出し先）がレベル間で矛盾していないか

## レビューの進め方

2段階でレビューする。全レベルを一気に作成した後にまとめてレビューすると、修正の連鎖が大きくなる。

1. **レベル内の正確性** — 各レベル作成直後に、ソースコードとの照合を行う
2. **レベル間の整合性** — 全レベル完成後に、上記チェックリストを実施する
